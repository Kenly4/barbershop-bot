# 💈 Telegram-бот записи в барбершоп

Бот, который полностью закрывает онлайн-запись клиентов: от выбора услуги до напоминания о визите. Владелец управляет графиком, услугами и ценами прямо из Telegram — без программиста.

> **Стек:** Python 3 · aiogram 3.x (async) · SQLite · APScheduler
> **Формат:** готовое решение под ключ, легко адаптируется под салон красоты, СТО, репетитора, клинику и любой бизнес с записью по времени.

## ✨ Возможности

### Для клиента
- Приветствие и понятное меню на кнопках
- Запись за 6 шагов: **услуга → дата → время → имя → телефон → подтверждение**
- Показываются **только реально свободные слоты** (без двойных броней и записи «в прошлое»)
- Кнопка «Поделиться контактом» + валидация телефона и имени
- Раздел «Мои записи»: просмотр, **самостоятельная отмена и перенос**
- Автоматическое **напоминание** о визите заранее

### Для администратора (владельца)
- ⚙️ **Панель `/settings` на кнопках — без единой строчки кода:**
  - график по дням недели (рабочий/выходной, часы работы)
  - шаг записи (по длительности услуги или фикс. 15/20/30/40/45/60 мин)
  - услуги и цены: добавить / переименовать / изменить цену и длительность / удалить
- 🔔 Мгновенное уведомление о новой записи с кнопками **Подтвердить / Отклонить / Заблокировать**
- Уведомления об отмене и переносе клиентом
- Списки записей: `/today`, `/upcoming`, `/date ГГГГ-ММ-ДД`

### Защита от спама
- Троттлинг частоты действий (middleware)
- Лимит активных записей на пользователя
- Валидация ввода и блокировка недобросовестных пользователей

---

## 🧠 Что внутри интересного (инженерная часть)

- **Асинхронная архитектура** на aiogram 3 (роутеры, FSM, middleware).
- **Умная генерация слотов** с учётом пересечений: запись на услугу 40 мин в 10:00 автоматически убирает слот 10:30 у следующего клиента.
- **Защита от двойной брони** на двух уровнях: повторная проверка перед вставкой + уникальный индекс в БД (корректно работает даже при гонке двух клиентов).
- **Настройки в БД, а не в коде** — график и услуги меняются на лету.
- **Планировщик** (APScheduler) для напоминаний, идемпотентный (не шлёт дважды).
- Чистая модульная структура: слой БД / бизнес-логика / хэндлеры / клавиатуры разделены.

## 🚀 Запуск за 3 минуты

```bash
python -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env              # впишите BOT_TOKEN и ADMIN_ID
python bot.py
```

- `BOT_TOKEN` — получить у [@BotFather](https://t.me/BotFather)
- `ADMIN_ID` — свой Telegram ID (например, у [@userinfobot](https://t.me/userinfobot))

## ⚙️ Настройка

График, шаг записи и услуги настраиваются **прямо в боте** через `/settings` (хранятся в SQLite, меняются на лету).

В `config.py` — только стартовые значения и общие параметры:

| Параметр | Назначение |
|---|---|
| `work_hours` | стартовый график (засеивается при первом запуске) |
| `DEFAULT_SERVICES` | стартовый список услуг |
| `DEFAULT_SLOT_STEP` | стартовый шаг записи (0 = по длительности услуги) |
| `booking_days_ahead` | на сколько дней вперёд открыта запись |
| `reminder_hours_before` | за сколько часов напоминать |
| `throttle_seconds`, `max_active_bookings` | параметры антиспама |

## 🗂 Структура проекта

```
bot.py                     — точка входа
config.py                  — настройки и стартовые значения
database/db.py, repo.py    — схема и запросы SQLite
services/slots.py          — генерация свободных слотов (ядро логики)
services/scheduler.py      — напоминания (APScheduler)
middlewares/throttling.py  — антиспам
states/                    — FSM-состояния диалогов
keyboards/                 — inline-клавиатуры
handlers/                  — логика: common, booking, my_bookings, admin, settings
utils/                     — валидаторы и форматирование
scripts/seed_demo.py       — демо-данные для презентации
```
